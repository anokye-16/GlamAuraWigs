Non-Functional Requirements

Quality attributes the system must satisfy
⚡

Performance

Pages load in under 2 seconds. Cart and price calculations are instant (client-side JS). Images are lazy-loaded to reduce initial load.

🔐

Security

Passwords hashed using PHP password_hash(). Form validation on client and server. HTTPS recommended for deployment. Input sanitization in PHP.

📱

Usability

Intuitive navigation with consistent design language. Toast notifications for all actions. Responsive layout adapts to mobile, tablet, and desktop screens.

📈

Scalability

MySQL database with normalized schema scales easily. Modular HTML/PHP structure allows new features to be added per module without breaking existing ones.

🔧

Maintainability

Shared CSS file (shared.css) across all pages. Shared navbar (navbar.html/js/css). Structured assignment folders for clear project organisation.

🛡️

Reliability

LocalStorage fallback for cart/auth state. Error messages shown on failed actions. Order IDs are unique per session. Form validation prevents empty submissions.
