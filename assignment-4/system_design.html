The system follows a standard 3-tier web architecture: the presentation layer (HTML/CSS/JS in the browser), the application layer (PHP on the server), and the data layer (MySQL database). The admin dashboard communicates directly with all back-end modules.

Table Relationships

  • users → orders: One user can place many orders (1:N)
  • orders → order_items: One order contains many items (1:N)
  • products → order_items: One product can appear in many orders (1:N)
  • users → feedback: One user can submit many feedback records (1:N)
  • suppliers → products: One supplier can supply many products (1:N)

4. Back-End Module Design

Structure of each business operations module

👥 CRM Module

  • Customer database — view all registered users
  • Customer communication records — query log
  • Feedback management — submit/view/resolve
  • Customer segmentation (VIP, Loyal, Regular, New)
  • Retention rate and satisfaction KPIs

💰 Finance Module

  • Payment processing (MoMo, Card, Bank simulation)
  • Revenue, expense, and profit KPIs
  • Invoice generation with unique ID
  • Invoice list — paid/unpaid tracking
  • Monthly revenue bar chart

📦 Ordering System

  • Product management — list all 24 products
  • Inventory management — stock levels with alerts
  • Order list with status (Pending → Delivered)
  • Purchasing system — supplier order creation
  • Delivery management — assign riders, track status

🚚 Supply Chain Module

  • E-market integration concept — supplier → warehouse → customer
  • Supplier directory with contact and reliability scores
  • Distribution tracking — shipment IDs and ETAs
  • Partner management — active/inactive suppliers
  • Supply chain flow diagram (visual)
