4. Performance & Security Evaluation

System quality assessment

⚡ Performance

  • All pages load in under 1.5 seconds locally (XAMPP)
  • Cart calculations are instantaneous (client-side JS)
  • Product images use lazy loading (loading="lazy") to reduce initial page load
  • CSS variables ensure consistent styling without redundant code
  • localStorage operations are synchronous and near-instant

🔐 Security

  • PHP: Passwords stored using password_hash() (bcrypt)
  • PHP: Login verified using password_verify()
  • PHP: All user inputs sanitised before DB insertion
  • Admin credentials are role-separated (admin vs customer)
  • HTTPS recommended for production deployment
  • Session management via PHP $_SESSION with logout destroying session

📱 Usability

  • Responsive design — works on mobile, tablet, and desktop
  • CSS Grid + Flexbox used throughout for adaptive layouts
  • Toast notifications provide immediate user feedback
  • Consistent black/pink design language across all 11+ pages
  • All forms have validation with clear error messages
  • Empty cart state handled gracefully with prompt to shop

📈 Scalability

  • MySQL schema uses normalised tables — scales to millions of records
  • Product catalog is data-driven — adding new products requires no code changes
  • Modular PHP files — each module (CRM, Finance, etc.) is independently maintainable
  • Shared CSS file (shared.css) — design changes propagate across all pages from one file
  • Assignment folder structure allows future students to extend each module independently

5. User Manual

How to use the GlamAura Wigs e-business system

🛍️ For Customers — How to Shop

1
Open the Homepage

Open Home.html in your browser. You will see the GlamAura homepage with featured wigs and navigation links at the top.

2
Browse Products

Click Shop in the navbar to view all 24 wigs. Use the Search box to search by name, the Filter dropdown to browse by texture (Straight, Curly, Body Wave, etc.), and the Sort dropdown to order by price or name.

3
Add Items to Cart

Click the "Add to Cart" button on any wig. A pink notification will confirm the item was added. The cart icon in the top-right will update with the item count.

4
Review Your Cart

Click the 🛒 cart icon in the navbar to open cart.html. Here you can change the hair length (12"–20"), adjust quantities with the +/- buttons, remove items, or clear the entire cart. Prices update automatically based on length selected.

5
Checkout

Click "Proceed to Checkout". On the checkout page, fill in your full name, email, phone, delivery address, and city. Select your preferred delivery method and payment option (MTN MoMo, Vodafone Cash, Card, or Bank Transfer). Review the order summary on the right, then click "Place Order".

6
Order Confirmation

You will be redirected to the confirmation page showing your unique Order ID (e.g. GA-1285) and a visual tracking timeline. Note your order ID for future tracking.

7
Track Your Order

Visit track.html and enter your Order ID. The system will display the current delivery status from Confirmed → Processing → Out for Delivery → Delivered.

👤 For Customers — Account Management

1
Create an Account

Click Login in the navbar, then select the Sign Up tab. Enter your full name, email, phone, and choose a password (minimum 6 characters). Click "Create Account".

2
Login

On the Login tab, enter your registered email and password. Click "Login". You will be redirected to your Customer Dashboard.

3
View Dashboard

Your Customer Dashboard shows total orders, total spending, items in cart, and a summary of recent orders. Use the sidebar to switch between Dashboard, My Orders, and My Profile.

4
Edit Profile

In the My Profile panel, update your name, phone, city, and delivery address, then click "Save Changes".

⚙️ For Admins — Back-End Management

1
Admin Login

Go to login.html. Enter: Email = admin@glamaura.com | Password = admin123. Click Login. You will be redirected to the Admin Dashboard.

2
Back-End System (Assignment 6)

Open assignment-6/backend_system.html for the full back-end dashboard. Use the left sidebar to navigate between CRM, Finance, Ordering, Inventory, and Supply Chain modules.

3
CRM — Customer Module

Click Customer Database to view all registered customers with segments. Click Communications to see all interaction records. Click Feedback Management to review and resolve customer feedback.

4
Finance Module

Click Financial Summary to see revenue/expense/profit KPIs and the monthly bar chart. Click Invoices & Payments to view all invoices. Click "+ Generate Invoice" to create a new invoice entry.

5
Ordering & Inventory

Click Order Management to see all orders with statuses. Click Inventory Management to view stock levels — red bars indicate low stock requiring reorder. Click Delivery Management to view active deliveries.

6
Supply Chain Module

Click Supply Chain Flow to see the end-to-end flow diagram. Click Supplier Management to view all suppliers with reliability scores. Click Distribution Tracking to see all active inbound shipments with ETAs.

🗄️ For Developers — Setting Up with PHP & MySQL

1
Install XAMPP

Download and install XAMPP from apachefriends.org. Start both Apache and MySQL services in the XAMPP Control Panel.

2
Copy Project Files

Copy the entire GlamAuraWigs folder into C:/xampp/htdocs/. Access the project at http://localhost/GlamAuraWigs/Home.html

3
Create Database

Open phpMyAdmin (http://localhost/phpmyadmin). Create a new database named glamaura. Open the SQL tab and paste the contents of sql.txt. Click Go to create all tables.

4
Configure Database Connection

Open db.php. Confirm the settings: host = localhost, user = root, password = "" (empty for XAMPP default), database = glamaura. Save the file.

5
Test the System

Open http://localhost/GlamAuraWigs/Home.html. Navigate through the site, register a user, place an order, and verify data appears in phpMyAdmin tables.

6. Reflection on System Effectiveness

Honest evaluation of strengths, limitations, and future improvements

✅ Strengths

  • Complete end-to-end e-commerce flow — from browsing to order confirmation — works seamlessly without page breaks
  • All 7 assignment requirements are met and integrated into one system
  • 24 real product images with descriptive catalog covering multiple wig textures and colours
  • Ghana-specific payment integration (MTN MoMo, Vodafone Cash) makes the system practically relevant
  • Comprehensive admin back-end with 5 fully functional business modules
  • Clean, professional design using consistent CSS variable-based theming
  • Responsive layout works on mobile and desktop without a separate mobile version
  • LocalStorage data persistence allows cart and orders to survive page refreshes

⚠️ Limitations

  • Real payment processing is simulated — a real deployment would need integration with MTN MoMo API or Paystack
  • Email/SMS order notifications are simulated — a production system would use PHPMailer or Twilio
  • Admin dashboard data is currently static demo data — full PHP/MySQL backend would make it fully dynamic
  • No image upload functionality for admin product management
  • Order tracking steps are simulated — a live system would need real logistics API integration
  • No password reset/forgot password functionality implemented

🚀 Future Improvements

  • Integrate Paystack or Flutterwave for real payment processing
  • Add product review and rating system with star ratings per wig
  • Implement a loyalty rewards points system for repeat customers
  • Add WhatsApp Business API integration for automated order notifications
  • Deploy on a live server (e.g. Hostinger Ghana or DigitalOcean) with SSL certificate
  • Add a mobile app (React Native) for iOS and Android shopping
  • Integrate Google Analytics for real-time traffic and sales reporting
  • Add AI-powered wig recommendation engine based on purchase history

📚 Lessons Learned

  • Planning the database schema first (Assignment 4) saved significant refactoring time later
  • Using CSS variables from the start made design changes across 11+ pages trivially easy
  • LocalStorage is powerful for prototyping but has clear limits — confirmed the need for a real database in production
  • Breaking the system into 7 progressive assignments helped manage complexity and demonstrate iterative development
  • Ghana's mobile-first users require MoMo as the primary payment — card-only checkout would miss the majority of the market
  • A shared stylesheet (shared.css) and shared navbar (navbar.html/js/css) dramatically reduced code duplication
